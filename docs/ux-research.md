# UX research

Use this document for **your** observations and study notes. The maintainers do not pre-fill conclusions here.

## How to use this file

Add dated sections when you run a quick study or interview (even informal). Keep entries short: **context**, **what you tried**, **what worked**, **what failed**.

## Prompts (answer in your own words)

1. **Who** is the primary user of your Leader Key setup (just you, a team, beginners vs power users)?
2. **What** is the main job-to-be-done: app launching, window management, media, dev workflows?
3. **Where** does the leader key break down (permissions, conflicts, cognitive load, too many groups)?
4. **When** do you reach for Leader Key vs Spotlight, Raycast, or the Dock?
